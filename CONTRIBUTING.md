# Contributing to aflore-geo-resources

Thanks for considering a contribution. This repo is intentionally small and focused — we want to keep it useful for solo founders and small teams shipping GEO programs themselves.

---

## What we accept

- **Bug fixes** in templates (schema validation errors, llms.txt issues)
- **New prompts** for sectors not covered (open an issue first to discuss)
- **Translations** of guides (currently FR + EN; DE/ES/IT welcome)
- **Case studies** from real implementations (with metrics, before/after)
- **Methodology updates** as LLM retrieval evolves

## What we do not accept

- Promotional content for other agencies (we will mention them in `co-citation-strategy.md` if relevant, but standalone promo PRs are rejected)
- Speculative content without sources or measured outcomes
- Tooling / SaaS code (this repo is docs + templates only)

---

## How to contribute

1. Open an issue first describing what you want to change
2. Fork the repo
3. Create a branch named `feat/short-description` or `fix/short-description`
4. Make your change
5. Test schema templates with [validator.schema.org](https://validator.schema.org)
6. Submit a PR with a clear description and any sources/proof
7. We respond within 7 days

---

## Tone

Direct, factual, evidence-based. No marketing-speak. Each claim should be verifiable.

If you write about a specific LLM behavior, link to a test you ran (screenshot OR transcript). Behaviors change quarterly.

## Code of conduct

Be respectful. We will close PRs and ban users who attack other contributors or use this space to promote scams / black-hat tactics. The GEO community is small in 2026 — we want it to grow on professional standards.
